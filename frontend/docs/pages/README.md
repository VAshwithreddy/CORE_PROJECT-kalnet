# Page Documentation

These docs explain how each route area should work.

Use the page doc that matches your task:

- `public-auth.md`: landing, login, forbidden.
- `employee.md`: employee home, my work, profile, requests, notifications.
- `department.md`: department home, team, projects, assignments, planner, blockers, digest.
- `executive.md`: overview, departments, portfolio, risks, digest, reports.
- `work-admin.md`: home, intake, routing, department heads, approvals, escalations, audit.
- `system-admin.md`: users, roles, permissions, service accounts, audit, settings.

Every page must include:

- Role shell.
- Page header.
- Main content area.
- Loading state.
- Empty state.
- Error state.
- Permission denied state.
- Mobile behavior.
- Keyboard-accessible primary workflow.
