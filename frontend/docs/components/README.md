# Component Documentation

These docs explain how shared frontend components should work.

Use these before building a page:

- `shells.md`: role layout shells.
- `data-table.md`: tables, filters, sorting, pagination, batch actions.
- `detail-drawer.md`: record details and editing drawers.
- `planner-board.md`: planning board and capacity UI.
- `approval-inbox.md`: approval queue and decision workflow.
- `audit-timeline.md`: audit event history.
- `command-palette.md`: quick navigation and search.
- `status-and-feedback.md`: badges, metric cards, empty states, errors, loading.

Shared components should live in `frontend/src/components/`.

Feature-specific components should live under the related `frontend/src/features/` folder.
